React Fast Training - User Booking Flow

1. Initial User Journey

User lands on reactfasttraining.co.uk
Browses course pages (First Aid at Work, Paediatric First Aid, etc.)
Clicks "Book a Course" or navigates to Booking page
Views Calendar with available dates
Can filter by course type or view all available sessions
Selects a specific course date and time

2. Booking Options (User Choice)

Modal/Page shows two booking options

Option A: Direct Online Booking

Click "Book Now Online"
Fill booking form:
  • Personal details
  • Number of attendees
  • Special requirements
  • Questions (optional)
Proceed to payment (Stripe)
Booking confirmed instantly
Receive confirmation email
Questions answered within 24hrs if provided

Option B: Booking Inquiry Form

Click "Make an Inquiry"
Fill inquiry form:
  • Name & contact details
  • Number of people
  • Preferred date/time
  • Questions about course
  • Payment preference
Submit form (no payment)
Email sent to instructor
Instructor responds with:
  • Answers to questions
  • Availability confirmation
  • Booking instructions

3. Inquiry Follow-up Process

After instructor responds to inquiry

Instructor Books

Instructor manually creates booking
Sends payment instructions
(Bank transfer, cash on day, etc.)

User Books Online

User returns to website
Completes online booking
⚠️ Risk Alert: Potential for double booking if both instructor and user try to book simultaneously.

Mitigation Strategies:
  • Inquiry system temporarily "holds" the slot for 48 hours
  • Clear communication in emails about who will complete booking
  • Real-time availability updates in the system
  • Unique inquiry reference number to track status

4. Email Communications

Inquiry Email to Instructor Contains:

  • User name and contact details
  • Course type requested
  • Date and time preference
  • Number of attendees
  • Any questions or special requirements
  • Preferred payment method
  • Inquiry reference number

Instructor Response Template:

Subject: Re: Course Inquiry - [Course Name] on [Date]

Dear [Name],

Thank you for your inquiry about our [Course Name] on [Date].

[Answers to specific questions]

I can confirm availability for [number] people on this date.

To secure your booking, you can either:

  • Book online at [booking link] using reference: [INQUIRY-REF]
  • Reply to this email confirming, and I'll reserve your places

Payment options: [Online/Bank transfer/Cash on day]

Please confirm within 48 hours to secure your places.

5. Technical Implementation Notes

Frontend Requirements:

  • Modal/page showing both booking options after course selection
  • Separate forms for direct booking vs inquiry
  • Clear visual distinction between options
  • Inquiry form should be simpler (no payment fields)

Backend Requirements:

  • New API endpoint: POST /api/bookings/inquiry
  • Inquiry status tracking (pending, responded, converted, expired)
  • Email service enhancement for inquiry notifications
  • Temporary slot reservation system (optional)
  • Inquiry reference generation
