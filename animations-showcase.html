Implementation Guide for React

Using Framer Motion (Recommended)

// Install: npm install framer-motion import { motion } from 'framer-motion'; // Fade In Animation const FadeIn = ({ children }) => ( <motion.div initial={{ opacity: 0, y: 20 }} animate={{ opacity: 1, y: 0 }} transition={{ duration: 0.6 }} > {children} </motion.div> ); // Hover Scale Animation const HoverCard = ({ children }) => ( <motion.div whileHover={{ scale: 1.05 }} transition={{ type: "spring", stiffness: 300 }} > {children} </motion.div> ); // Stagger Children Animation const StaggerContainer = ({ children }) => ( <motion.div initial="hidden" animate="visible" variants={{ visible: { transition: { staggerChildren: 0.1 } } }} > {children} </motion.div> );

Page-Specific Implementations

// Homepage Hero Animation const HeroSection = () => ( <motion.div initial={{ opacity: 0 }} animate={{ opacity: 1 }} transition={{ duration: 1 }} > <motion.h1 initial={{ y: -50 }} animate={{ y: 0 }} transition={{ delay: 0.2, type: "spring" }} > React Fast Training </motion.h1> </motion.div> ); // Course Card Animation const CourseCard = ({ course }) => ( <motion.div whileHover={{ y: -5 }} transition={{ duration: 0.2 }} className="course-card" > {/* Card content */} </motion.div> ); // Loading State const LoadingScreen = () => ( <div className="loading-screen"> <div className="pulse-heart" /> <p>Loading your training experience...</p> </div> );

Performance Tips

  • Use CSS animations for simple effects (better performance)
  • Implement lazy loading for animation libraries
  • Use will-change CSS property sparingly
  • Prefer transform and opacity for animations (GPU accelerated)
  • Reduce animation on mobile devices
  • Honor prefers-reduced-motion for accessibility
// Respect user's motion preferences @media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; } }
